# Plant Cards — Phase A: Data Model & Form Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the schema fields, vocabularies, validations, API endpoints, and admin form inputs needed to capture every piece of data shown on the printable plant card. After this phase, an admin can fill in all card-relevant data in `SpeciesFormModal` and round-trip it through the API. The card view itself is built in Phase B.

**Architecture:** Add ~15 nullable columns to `plant_species` via one migration. Centralize controlled vocabularies as Ruby constants on `Plant::Species`. Extend `Api::V1::PlantsController#filter_options` to expose the new vocabularies; extend `species_params` (input) and `serialize_species` (output). Add a new `conception` section to `SpeciesFormModal.tsx` with chip-group/select inputs for every new field.

**Tech Stack:** Rails 8.1, ActiveRecord, PostgreSQL JSONB, Minitest, React 18 + TypeScript via Inertia, Tailwind 4.

**Spec reference:** `docs/superpowers/specs/2026-05-06-plant-cards-design.md` — sections "Data Model", "Strate / Cycle / Successional palettes", "Vocabularies", "Hardiness conversion".
